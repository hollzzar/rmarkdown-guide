R Markdown for Psychology Graduate Students

Chapter 7 Tips

This chapter contains some general tips and information to help you with R Markdown. These are some general keys to success with R Markdown (as well as with R itself):

  • Treat your data as read-only
  • Comment code early and often
  • Keep code chunks small
  • Label chunks for diagnosing issues
  • Nest all files under one directory (if possible)

7.1 General information

7.1.1 Special characters

If you want any characters that are special for R Markdown, \(\LaTeX\), or pandoc to appear as text, rather than performing some function, you need to “escape” them with a backslash \. For example, pound signs/hashtags, backslashes (as in the previous sentence), and dollar signs need to be preceded by a backslash. This also applies to any chunk output containing strings. For example, I often run into this with knitr::kable tables, particularly using \(\LaTeX\) output format, when I’m using \(\LaTeX\) functions or characters.

Speaking of \(\LaTeX\), you can engage “math-mode” by putting dollar signs around \(\LaTeX\) math commands. This way, you can include equations, IPA symbols, and the like in R Markdown (even if you’re not outputting to a PDF). For example, I can write \(e = mc^2\) in a sentence like this just by wrapping the equation in a single set of dollar signs, or I can use two sets to center the equation: \[e = mc^2\]

7.1.2 Calling R in text

A very useful aspect of R Markdown is that you can call R objects and functions in markdown/YAML by sandwiching them between backticks. For example, let’s say I want to look at the names of the species in the iris dataset.

# Pull species column from iris and get unique values in column
species <- iris %>% pull(Species) %>% unique()

# Print species variable
print(species)
## [1] setosa     versicolor virginica 
## Levels: setosa versicolor virginica
# Get number of unique species
species_count <- length(species)

# Print number of species
print(species_count)
## [1] 3

The variable species_count has the number of unique species names in the dataset. Just as in a code chunk, I have to specify that I’m working with R code by including a lower-case r in the backticks with the variable. So if I put “r species_count” between backticks, I can write that there are 3 species. Or I can pull the list by referencing the species variable: setosa, versicolor, virginica.

7.1.3 Markdown

R Markdown type-setting functions include italics, bold, and strike-through. There are also special features like (parentheses), [square brackets], and “quotation marks”. With all of these, if you highlight the text you want to wrap in these characters (by clicking and dragging your cursor), you can just hit the character once to enclose the text automatically. This way, you don’t have to go to the beginning and end of the text and place the characters individually.

Another note is that line spacing matters in R Markdown. For example, if I want to include bullet points after this sentence, they won’t render properly if I don’t hit enter twice before starting them. In other words, I need to have a full line of white space before bullet points and numbered lists. If you’re having issues with your document rendering correctly, make sure you have line breaks between lists, paragraphs, and headers.

7.4 Updating software/packages

You may get warnings or errors related to packages being out of date or certain functions being “deprecated.” You may need to update packages and software that interacts with R and R Markdown to get your code to run and documents to knit. Depending on the feature that is out of date, there are different strategies for updating.

  • Update your TeX distribution from the command line
  • Update all packages (including rmarkdown) in library with update.packages(path)
  • Update individual packages by reinstalling with install.packages(package)
  • Update R in the console with updateR package
  • Redownload RStudio to update
